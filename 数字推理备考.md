## 各章节详细教学内容

### 第1章：How to ACE Numerical Reasoning Tests（总体应试策略）

数字推理是与数字相关的批判性思维，主要测试数字的：
- 代表性形式（数列模式）
- 单个数字字符

**测试题型包括：**
- 数列规律（Number series patterns）
- 图形中的数字组合
- 矩阵中的数字
- 文字题（单变量/多变量/速率/未知数）

**应试策略（3步法）：**
1. **识别题型** — 先判断属于哪类问题
2. **快速能解则解** — 若能快速完成，立即作答
3. **不能解则跳过** — 先跳过，最后用猜的方式填答

**主要章节模块：**
- 模式识别（上升/下降/组合/间隔/替代/跳跃）
- 图形与矩阵数字
- 文字题编码求解
- 代数求缺失数

---

### 第2章：Numerical Reasoning Pattern Detection - Up（上升型数列）

**题型识别：**
- 数列中每个后续值都高于前一值
- 例：`2, 4, 6, ?` 或括号配对：`[16, 33] [7, 15] [?, 55]`

**解题策略：**
1. 识别数字之间的关系（规律对所有数字一致）
2. 用规律找出缺失数

**寻找规律的规则：**
- 数字间隔较小 → 考虑加法
- 数字间隔较大 → 考虑乘法
- 加法/乘法无效 → 考虑两者组合

**关键规则：**
- 先从加法/乘法入手
- 若最简单的运算符不适用，尝试组合运算
- 验证规律：用规律核对所有已知数字对

---

### 第3章：Numerical Reasoning Pattern Detection - Down（下降型数列）

**题型识别：**
- 数列中每个后续值都低于前一值
- 例：`4, 2, 0, ?` 或 `[16, 8] [14, 7] [?, 11]`

**解题策略：**
与上升型相同，但运算方向相反：
- 数字间隔较小 → 考虑减法（有时也用于较大间隔）
- 数字间隔较大 → 考虑除法
- 减法/除法无效 → 考虑两者组合

**关键规则：**
- 先从减法/除法入手
- 若不适用，尝试组合运算
- 始终验证规律是否适用于所有数字

---

### 第4章：Numerical Reasoning Pattern Detection - Combinations（组合型数列）

**题型识别：**
- 数值既有上升也有下降（交替变化）
- 例：`4, 2, 6, 3, ?`（÷2, ×3, ÷2, ×3…）
- 特征：一般有某种交替形式（高、低、高、低…）

**解题策略：**
1. 识别数字间的关系
2. 识别**多组规律**共存
3. 找出每组规律分别应用于哪些位置（奇数位、偶数位等）

**关键挑战：**
- 需要同时处理多个规律
- 先找一个子规律，再找另一个子规律
- 交替型规律最常见：偶数位用一种规律，奇数位用另一种

---

### 第5章：Harder Numerical Reasoning Patterns 1 - Gaps（进阶：间隔型/指数型）

**题型识别：**
- 数字间隔非常大，无法用简单四则运算直接找到规律
- 例：`1, 4, 9, 16, 25, 36`（平方数列）
- 或括号配对：`[2, 4] [11, 121] [?, 36] [4, 16]`（平方关系）

**解题策略：**
1. 先尝试基础运算（+、-、×、÷）
2. 若失败，考虑**指数（exponentials）**：平方、立方
3. 若失败，考虑**根号（surds/roots）**：平方根、立方根

**规则口诀：**
加法/乘法 → 指数 → 根号 → 组合

---

### 第6章：Harder Numerical Reasoning Patterns 2 - Stand-in（进阶：替代数字）

**题型识别：**
- 数列中存在固定重复的"占位数字"（常数/哑数）
- 目的是干扰规律识别
- 例：`2, 4, 2, 8, 2, 12, 2, ?` — 其中"2"是替代数字

**解题策略：**
1. 找出数列中**重复出现**的数字
2. 将其排除，只看剩余数字
3. 在剩余数字中寻找规律
4. 用规律找出答案

**关键规则：**
- 看不出规律时，检查是否有"替代数字"
- 识别并排除后，用之前学的规律方法分析

---

### 第7章：Harder Numerical Reasoning Patterns 3 - Jumps（进阶：跳跃型/双轨规律）

**题型识别：**
- 数列中有**多个规律**，每个规律只适用于间隔的数字（第1、3、5位用一种规律；第2、4、6位用另一种规律）
- 识别标志：没有明显的逐步规律，通常是交替数字
- 例：`5, 0, 7, -1, 9, -2, 11, -3`（奇数位+2，偶数位-1）

**解题策略：**
1. 尝试基础运算，若有两种方向同时存在则考虑跳跃
2. 把数列分成两组（奇数位、偶数位）
3. 分别找每组的规律
4. 用各自规律找出缺失数

---

### 第8章：Number Patterns - Grouped in Images（图形分组数字规律）

**题型识别：**
- 数字被组合在图形中（表格、圆形、方框）
- 例：`21 is to [3,5,4/2,6,1] as 42 is to [6,10,8/4,8,?]`
- 本质仍是数列规律，但有多个"验证点"

**解题策略：**
1. 识别数字之间的关系
2. 利用多个组合**验证规律**（交叉检验）
3. 用确认的规律找出缺失数

**关键点：**
- 不同于简单数列——有多个参考组供验证
- 规律通常不像预期那么直接，需要多点验证

---

### 第9章：Number Patterns - Matrices（矩阵数字规律）

**题型识别：**
- 数字以行列矩阵形式排列
- 需要找出缺失值
- 既要考虑**数字规律**，又要考虑**方向规律**（横向、纵向、对角线、甚至跳跃）
- 识别标志：数字排列在行和列中

**矩阵示例：**
```
| * | 35 | 23 |
| 25 | 15 |  7 |
| 16 |  8 |  ? |
```

**解题策略：**
- 横向看规律
- 纵向看规律
- 对角线看规律
- 找到能贯穿整个矩阵的一致规律

---

### 第10章：Pattern Detection within Groups（组内字符规律）

**题型识别：**
- 规律存在于**组内字符本身**，而非组与组之间
- 例：`{ 22=4 } { 41=5 } { 15=6 } { 14=? }`（组内字符相加/相乘等）
- 与普通数列不同——不是纵向/横向加减，而是对组内每个数字进行运算

**解题策略：**
1. 先看**结果数字**，逆向推导可能的运算
   - 如 4 = 2×2，或 4 = 2+2
2. 用其他组**验证规律**
   - 如 5 = 4+1（两位数 41 的各位相加）
3. 确认规律后，应用到未知组

**综合提示：**
- 同时应用之前所有章节的技巧（加法/乘法/指数/替代等）
- 先从结果逆推，再交叉验证

---

### 第11章：Coding Worded Problems - An Overview（文字题编码总览）

**题型识别：**
- 用文字描述数学关系
- 例：Rose 每12分钟写3页，Eric 每30分钟写12页，同时写20分钟后谁多写了几页？
- 可能涉及：分数/距离/未知变量等

**通用解题步骤（5步法）：**
1. **找到问题要求**（The Requirement）
2. **速记信息**（Notate）— 用简写或图示记录关键数据，排除干扰
3. **运算**（Calculate）— 将符号化信息与问题建立联系
4. **得出答案**（Solve）
5. **检验**（Check）— 验证答案是否合理

**编码（Coding）的核心理念：**
> 将文字题转化为数学符号/公式，消除文字干扰，专注于数学关系

---

### 第12章：Coding and Solving Worded Problems - Working with Information（信息处理）

**题型特点：**
- 较长的文字题，可能需要单位换算
- 涉及多个分数、不同单位（mL/L、cm/m等）

**解题策略：**
- 遵循总体5步法
- 关键差异：**统一单位**（苹果对苹果原则）
  - 若有 mL 和 L，先将其中一个换算成另一个单位

**一般步骤：**
1. 找到要求
2. 阅读并提取信息（短注）
3. 统一单位
4. 计算
5. 验证

---

### 第13章：Coding and Solving - Finding the Unknown with 1 Variable（单变量求解）

**题型识别：**
- 要求找出某个未知量的值
- 关键词：`"how much..."` / `"how many..."`
- 例：一箱 4.7kg，另一箱 2.5kg，要从第一箱取出多少放入第二箱使两箱等重？

**解题策略：**
- 遵循5步通用法
- 额外步骤：**画图/可视化**问题情境（若不确定时特别有用）
- 设定未知量，建立等式

---

### 第14章：Coding and Solving - Finding the Unknown with Multiple Variables（多变量求解）

**题型识别：**
- 有**两个或多个未知量**需同时求解
- 例：两数之差为25，其中一数小于6，另一数能被6整除，求这两数

**解题策略：**
1. 找到问题要求
2. 设置**约束条件**来缩减选项范围
3. **试值法**：对一个变量试合理的值，再代入求另一个变量
4. 满足所有条件且在选项中的即为正确答案

---

### 第15章：Coding and Solving - Rates（速率类文字题）

**题型识别：**
- 两人/多人共同完成任务，求时间
- 例：Steve 独自粉刷围栏需30小时，Elizabeth 需15小时，两人合作需多久？
- 识别特征：任务完成时间、参与者数量、完成进度

**解题策略（速率单一化原则）：**
> 将速率降至"每单位时间完成多少"，然后再扩展计算

**步骤：**
1. 计算每人每单位时间的工作量
2. 合并速率（相加）
3. 用总要求除以合并速率

---

### 第16章：Twice as Much / "n" Times as Much（倍数关系题）

**题型识别：**
- 要求从文字中识别数学倍数关系
- 例：一个抽屉有88只袜子，白色是黑色的1/3，求黑色袜子数量
- 涉及：基本算术、小数、分数、百分比

**解题策略（视觉分析法）：**
1. **画出题目**（可视化关系）
2. 将已知关系转化为图示（如条状图/比例图）
3. 从图中直接读出答案或建立方程

**关键技巧：**
- 先画图，再写方程
- 图示帮助理解"1/3倍"等相对关系

---

### 第17章：Missing Numbers using Algebra（代数求缺失数）

**题型识别：**
- 最常见的数字推理题型
- 实质是"找未知量"，即广义代数
- 例：水果箱有10个水果：4个苹果，橙子是苹果的一半，其余是梨，求梨的数量
- 例：Tania 每小时薪酬，加10%奖金，6小时共赚$200，求时薪

**解题步骤（5步代数法）：**
1. **确定未知量**（What are the unknowns?）
2. **设变量**（Let x = …）
3. **建立方程**（Express total in algebraic form）
4. **解方程**（Solve for x）
5. **验证答案**（Check back against the problem）

**示例：**
- 6x + 0.6x = 200 → 6.6x = 200 → x = 30.30（Tania时薪$30.30）

**核心原则：**
> 代数就是找未知数——明确你在找什么！
> 列方程时注意时间紧迫，优先简化再求解

### 第18章：Revision & Final Exam（复习与期末考试）

**各章节记忆要点汇总：**

| 章节 | 核心规则 |
|------|----------|
| 上升型 | 加法/乘法，较大间隔用乘法 |
| 下降型 | 减法/除法，较大间隔用除法 |
| 组合型 | 分组，寻找多个子规律 |
| 间隔型(Gaps) | 指数/根号/组合 |
| 替代型(Stand-in) | 找出并排除常数，再找规律 |
| 跳跃型(Jumps) | 分奇偶位，各自找规律 |
| 图形分组 | 多点验证，交叉核实规律 |
| 矩阵 | 横向/纵向/对角线多方向检验 |
| 组内规律 | 从结果逆推，交叉验证 |
| 文字题 | 识别要求→速记→建模→计算→验证 |
| 代数 | 设变量→建方程→解→验证 |

**总体应试策略：**
> 识别题型 → 若可快速解答则立即解 → 否则跳过 → 最后猜答

**期末考试：**
- 标准版：30题，限时25分钟
- 进阶版（9年级入学）：50题，限时30分钟
