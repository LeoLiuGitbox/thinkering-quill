Checkpoint 1 — How to ACE Reading Comprehension Tests
What reading comprehension actually tests:

Subtleties in meaning (e.g. "furious" vs "angry")
How punctuation changes pace/meaning
How stories convey feelings
Purpose and characteristics of different text types
Literal vs figurative language
Understanding a text from different perspectives
The ACE Framework:

Find the question
Read with focus (know what you're looking for)
Eliminate options
Use bottom-up strategies (keywords, ranking, exclusion)
Time strategy:

Read question first → if quick, complete it; if not, skip and return
With 30 seconds left: complete if possible, otherwise guess B or C
Never spend more time than allocated per question
Common mistakes: Taking too long, misreading the question, having no strategy

Checkpoint 2 — Critical Test Strategies
The 6-step approach:

Read the question/s first
Identify text type (< 1 second)
Read passage quickly, targeting relevant areas
Do keyword/sentence checks against the question
Use an on-balance approach if required
Evaluate answer options → eliminate → choose
5 Types of Questions (with strategies):

Question Type	What it Tests	Strategy
Main-idea / purpose / tone	Overall point or feeling	Scan for keywords/phrases that lean toward one option
Fact/event-based	When/how/why something happened	Locate by identifying keywords in the question
Inference/implication	What's implied but not stated	Find keywords in passage that suggest an outcome
Sequence/order	Order of events	Use text structure/layout
Words/phrases in context	Word meaning within a passage	Read the full sentence/paragraph around the word
Focusing your reading:

Know the structure of each text type to locate information fast
Don't read parts you don't understand — focus on what the question asks
Don't read too slowly; only spend the allocated time
Checkpoint 3 — What's Certain and What's Not?
Absolute words (0% or 100%): always, never, impossible, every, all, none

Easy to disprove — only need 1 counterexample
Use as targets for quick elimination
Moderate words: sometimes, often, usually, can, may

Harder to disprove — safer answer options
In practice: When you see an absolute in an answer option, scan the passage for a single exception. If found → eliminate that option immediately.

Checkpoint 4 — Narrative: Literal
Parts of a Narrative (likened to Frozen in the course):

Setting — where and when
Theme — underlying moral/message
Characters — major and minor
Plot:
Problem
Response
Action
Result
Resolution — end of the problem
Why structure matters: Knowing where things appear in a narrative lets you locate answers quickly without re-reading the whole text.

Literal questions: The answer is stated in the text — find it by scanning with keywords from the question.

Checkpoint 5 — Narrative: Figurative
Figurative language types:

Simile — comparison using like or as (e.g. "swarmed out like a school of fish")
Metaphor — direct comparison (e.g. "the path home was a battlefield")
Hyperbole — exaggeration for effect
Personification — giving human traits to non-human things
Alliteration — repeated consonant sounds
Onomatopoeia — words that sound like what they describe
Key rule: Do NOT interpret figurative language literally — it will lead to the wrong answer.

Tone questions are common with figurative language. Use an on-balance keyword approach: gather all tone-related words across the passage and determine which mood they collectively lean toward.

Checkpoint 6 — Narrative: Implied
What "implied" means: The writer assumes you'll understand hints without stating them outright.

Keywords in questions that signal implied questions:

implied, suggest, ascertain, interpret
"how does [character] feel?"
"what can we infer?"
How to answer: Find hints (keywords/phrases) in the text that collectively lean toward one option over others. Like inferring someone likes a movie from posters in their room — it's never confirmed, but the evidence points there.

Checkpoint 7 — Non-Fiction: Provided
Non-fiction text types: Informative articles, factual information

Structure of non-fiction:

Factual/general information
Cause and effect relationships
Inferences (strongly implied but not stated)
Characteristics:

Lots of detail
Objective tone
Rarely contains figurative language
Detail questions (this checkpoint): Answer is directly in the passage. Often about cause-effect. Use text structure (headings, paragraphs) to locate the answer quickly.

Checkpoint 8 — Non-Fiction: Implied
What non-fiction implies: consequences, results, processes

Keywords signalling inference questions: implied, suggest, ascertain

Approach: Find hints within the words themselves (not just surface reading). Use background/contextual information within the passage. The answer won't be stated but will be strongly supported by the evidence.

Checkpoint 9 — Interviews
Structure:

Interviewer — asks questions to gather information
Interviewee — provides information/opinions
Purpose: Reveal profile of the person + their perspective on an issue

Question types in interview texts:

View/position of interviewer or interviewee
Detail questions (facts stated in the interview)
Inference questions (reading between the lines)
Attitude of the interviewer/interviewee
Key tip: The position (who is speaking) matters — distinguish between what the interviewer vs. interviewee believes.

Checkpoint 10 — Historical Texts
What it is: A story about a past event, written by a third party (not the subject)

Why it's different from autobiography: Written by someone other than the person in question — more like narration. It mixes fact and sometimes fiction because it draws on memory, interviews, and multiple sources.

Question types:

Inference questions
Detail questions (facts and information)
Figurative language questions
Checkpoint 11 — Autobiographical Texts
What it is: About a person's life, written by that person. Classified as non-fiction, but contains fictional elements.

Fictional elements that can appear:

Memory loss
Selective passages
Made-up stories / exaggerations
Personal interpretation of past events
Why people write autobiographies:

Come to terms with a major experience
Share learning with the public
Express identity and self-knowledge
Approach: Combined fiction + non-fiction strategy — look for both literal detail AND implied meaning/tone.

Checkpoint 12 — Poetry: Structure
What poetry is: Creates meaning in few words using rhythm and figurative language. Organised in stanzas; lines referenced by number.

Parts of a poem:

Subject matter / theme — what it's about; what ideas it contains
Technique — how the poet conveys the idea:
Structure — number of stanzas
Tone — voice/sound (funny, angry, sarcastic…)
Mood — feeling/atmosphere (serene, scary, troubled…)
Imagery — metaphors, symbols, personification
Rhythm — beat, repetition of words or vowels
Rhyme — matching sounds at end of lines
Why structure matters: Knowing these parts helps you identify what type of question is being asked and where to look.

Checkpoint 13 — Poetry: Meaning
Two levels of meaning:

High level — overall theme and commentary
Individual level — what is expressed line by line
Figurative vs literal: Figurative language requires more cognitive steps. E.g. "My hand is an ice block" requires you to process: hand → ice block → frozen cold → extreme cold. More work, but bigger emotional impact.

Mood: Comes from the collective effect of words. Steps to find it:

Understand individual word meanings (use a dictionary if needed)
Gather all adjectives, adverbs, and key nouns
Determine which mood they collectively lean toward
Key rules:

Poetry is mainly figurative — "what it says ≠ what it means"
Think about the overall theme first, then drill into individual lines
Most questions test figurative language
Checkpoint 14 — Cartoons: Imagery
What cartoons do: Mix humour with strong social/political messages using images, symbols, and text.

Visual elements to analyse:

Symbolism — objects/images standing for larger ideas
Exaggeration — making a point by over-representing something
Labels — clarifying the meaning of things in the image
Analogy — comparison between unlike things sharing a characteristic
Irony — the gap between how things are vs. how they should be
White space — what's absent is as important as what's present
Size/proportion — disproportionate elements signal importance
Position — placement of figures/objects carries meaning
Question focus: What is the cartoonist's perspective/point?

Checkpoint 15 — Cartoons: Language
Text in cartoons is usually short and figurative — adds to the social/political commentary.

How to analyse cartoon text:

Source — who/what is the text coming from? (Character speech, heading, caption?)
Style — how is it written?
CAPITALS / bold → strong, forceful statement
Curly/flowing font → creativity, flexibility
Break it down — split text into components, find connotations for each, then combine meaning
Because text is short: Every word carries more weight. Treat it like a compressed poem.

Checkpoint 16 — Punctuation
Tested in two ways:

Direct — find the sentence with correct punctuation
Indirect — find meaning or tone of a sentence (where punctuation heavily affects interpretation)
Punctuation you must know:

Mark	Purpose
Full stop	Ends a sentence; marks abbreviations
Comma	Pause; separates items in a list; joins clauses
Semicolon	Links two related independent clauses

Ellipsis (…)	Trailing thought; suspense; omission
Hyphen	Joins compound words
Checkpoint 17 — Proverbs
What proverbs are: Short sayings expressing an obvious truth. Words are often figurative — need wider interpretation.

3-step method to find meaning:

Identify key components (the main nouns/concepts)
How do they relate? (What is the natural relationship between these things?)
Extend to real life (What does this relationship mean as a life/human truth?)
Example: "A tree is known by its fruit"

Components: tree, fruit, "is known by"
Relationship: a tree produces fruit; the quality of fruit reflects the quality of the tree
Real life meaning: A person is judged by what they produce/do — their actions reveal their character

Persuasive Writing & Discussion Written Expression — Full Content Summary
18 Checkpoints | Covers: Persuasive writing (argue one side) + Discussion writing (both sides)

Checkpoint 1 — How to ACE Written Expression Tests
Key distinction:

Persuasive writing — persuade reader to adopt one side
Discussion writing — discuss both sides equally; opinion optional and not meant to persuade
The basics (expected but not what scores top marks):

Vocabulary, grammar, spelling, sentence structure
What actually wins marks:

Selection of arguments/discussion points — thoughtful, intelligent, unique?
Development of arguments — logical flow, convincing, relevant?
Scoring differentiation: Strong arguments + strong development = top scores. Weak arguments = lower scores regardless of expression quality.

Checkpoint 2 — How to Write Quickly Under a Time Limit
The 3-step process (time allocation):

Phase	Time %	Purpose
Plan	25%	Select and structure arguments/discussion points
Write	60%	Execute the plan using templates
Review	15%	Check expression only (not structure)
Time examples:

20 min exam → 5 min plan, 12 min write, 3 min review
30 min exam → 7.5 min plan, 18 min write, 4.5 min review
45 min exam → 11 min plan, 27 min write, 7 min review
Key reason for more planning time: Argument selection is the biggest scoring differentiator — it's worth the investment.

Checkpoint 3 — Language to Avoid & Tone
3 areas to get right:

1. Tone — avoid emotionally charged language:

✗ CAPITALISED LETTERS for emphasis
✗ Exaggerated claims ("this will mean the end of the world")
✗ Excess punctuation ("But that can't be true!!!!")
✗ Rhetorical questions ("But who would believe that?")
✓ Rely on well-reasoned arguments to persuade
2. Clear expression:

Unclear writing = reader can't follow your argument = lost marks
One idea per sentence; don't cram too much into one sentence
3. Avoid colloquialisms and contractions:

✗ "kids", "gonna", "don't", "it's"
✓ "children", "going to", "do not", "it is"
Formal writing projects authority and credibility
Checkpoint 4 — Essential Parts of a Persuasive & Discussion Piece
Persuasive Writing Structure (in order):

Introduction — topic sentence + stance + signpost 2 arguments
Argument 1 — 4 sentences: TEEL (Topic, Explanation, Example/Evidence, Link back)
Argument 2 — 4 sentences: TEEL
Counter-argument & reorientation (optional) — opposing view + direct rebuttal
Conclusion — summary of 2 arguments + insight (1 sentence)
Discussion Writing Structure (in order):

Introduction — topic sentence + state both sides + claims of both sides
Point 1 (Side A) — 4 sentences: TEEL
Point 2 (Side B) — 4 sentences: TEEL
Point 3 (optional) — additional point for either side
Opinion (if required) — only if the prompt asks for it
Conclusion — summary of both sides
Checkpoint 5 — Planning
Persuasive Writing Plan template:

Stance: Clear one-sentence position directly addressing the question
A1: Claim in one clear sentence answering "I support this because…"
A2: Different claim (not just rephrased A1) answering "I support this because…"
CA&R (optional): Counter-argument + reorientation
Conclusion: One sentence summarising A1 + A2
Discussion Writing Plan template:

Topic: Define the issue clearly
Side A — Point 1: Claim for side A
Side B — Point 2: Claim for side B
Opinion (if required)
Key planning rules:

Arguments must be distinct — different words with the same meaning = same argument
Each claim must clearly answer WHY you support the stance
Plan before you write — never start writing without a plan
Checkpoint 6 — Length
Why length matters:

Too much: More errors, irrelevant content, overwhelms reader
Too little: Arguments underdeveloped, reader left dissatisfied
Optimal length (from top-scoring pieces):

Persuasive: ~13–15 sentences (intro + 2×TEEL + optional CA&R + conclusion)
Discussion: ~13–17 sentences depending on structure
Rule: Use the structure as your guide to length — each part has a set number of sentences. Hit that target and you'll naturally land in the right range.

Checkpoint 7 — How to Come Up With Strong Arguments
Core insight: Arguments already exist in the issue — your job is to uncover and select them.

The lolly bag analogy: The issue is the bag; arguments for = red lollies; arguments against = black lollies. You pick from the bag — you don't invent lollies that aren't there.

The 3 Dimensions for generating arguments:

Dimension	Focus
Individual	How does this affect a person? (health, emotions, rights, finances)
Social/Community	How does this affect groups, communities, society?
Broader	How does this affect the environment, economy, nation, world?
Strength test for an argument:

Is it directly related to the issue?
Is it logical and defensible?
Can you support it with an example or evidence?
Is it distinct from your other argument?
Checkpoint 8 — Making Arguments Convincing (TEEL in Depth)
The 4-sentence TEEL progression:

Step	Label	What it does	Example
Sentence 1	T — Topic/Claim	State the argument	"Smoking should be banned because it is bad for a person's health."
Sentence 2	E — Explanation	Explain WHY the claim is true	"Cigarettes contain toxic chemicals such as tar and carbon monoxide which damage the lungs and cardiovascular system."
Sentence 3	E — Example/Evidence	Make it real and specific	"Studies show that long-term smokers are 25 times more likely to develop lung cancer than non-smokers."
Sentence 4	L — Link back	Connect back to your stance	"It is therefore clear that smoking poses a severe health risk, justifying a ban to protect citizens."
Critical principle: Each sentence must build on the previous one and answer any question the reader might have — no gaps in logic.

Common TEEL mistakes:

Explanation that restates the claim instead of explaining it
Example that is vague or loosely related
No link back to the stance/topic
Checkpoint 9 — How to Support Arguments (Evidence & Examples)
Checklist for good evidence/examples:

✓ Directly relates to the claim
✓ Best example available (not loosely related)
✓ Specific (not vague)
✓ No incorrect absolute statements
✓ No incorrect cause-and-effect claims
Types of evidence/examples you can use:

Past/historical experience — "History has shown that…"
Comparison — "A side-by-side study showed… / In 1900… while in 2000…"
Statistics — "It was reported that X% of…" (use carefully — can become outdated)
Expert opinion — "According to [field expert]…"
Personal/relatable example — "For example, a student who…"
Hypothetical scenario — "Imagine if…" (use sparingly)
Checkpoint 10 — Starting With a Strong Introduction
Persuasive intro (3 parts):

Topic sentence — brief background (1 sentence)
Stance — crystal clear, no ambiguity
Signposting — state the claim of your 2 arguments only (not conclusion/reorientation)
Discussion intro (3 parts):

Topic sentence — brief background (1–2 sentences)
State both sides of the issue
Provide the claims of both sides
Template (Persuasive) example:

"The cost of textbooks… is a sore point for many. Textbooks should be provided free to all students. This is because free textbooks reduce financial burden on families and enable equal access to education for all students."

Checkpoint 11 — Counter-Arguments & Reorientations (Persuasive only)
What it is: Acknowledge the opposing view, then immediately refute it.

Structure (2 sentences):

State a strong (not weak/vague) opposing argument — 1 sentence
Directly address and disprove it — 1 sentence
Templates:

"While some argue that [counter-argument], this is not the case because [reorientation]."
"Opponents may argue that [counter-argument], however [reorientation]."
"Admittedly, [counter-argument], but [reorientation]."
Risk warning: If done poorly it decreases overall quality. Only include if you can do it well. A high-quality CA&R can significantly boost your score.

Checkpoint 12 — Conclusions
How to write it: Summarise your 2 claims in 1 sentence.

Example:

"In conclusion, graffiti is not a form of art because it causes damage to property and is a bad influence on younger children."

What to avoid:

Introducing new information
Including too much information
Going off topic
Scoring note: Small marks available here, but easy to lose marks by including new information. Keep it clean and brief.

Checkpoint 13 — Common & Classic Exam Prompts
Common topics:
Junk food, school uniforms, homework, rubbish, country vs. city, sports, extended weekends, teachers, superhero powers, languages/curriculum, smoking, bullying, voting, and similar.

4-step prompt analysis process:

Read the question and circle key words
Identify — is it persuasive or discussion? What exactly is being asked?
Rephrase the question in your own words
Check your rephrasing against the original — does it match?
Key warning: Even on familiar topics, misreading the question is a top mistake. The question may have a specific angle (e.g. "Should junk food be banned in schools?" ≠ "Should junk food be banned?")

Checkpoint 14 — Uncommon & Specialised Exam Prompts
Examples of specialised topics: cloning, self-driving cars, whaling, corporate sponsorship, space exploration, AI, genetic engineering

Good news: These prompts almost always come with descriptive details and hints for arguments. Read them carefully.

Approach: Same 4-step process as common prompts + read the prompt more deeply for embedded hints/tips about possible arguments.

Use the 3 dimensions (individual, social, broader) to generate arguments even on unfamiliar topics — the dimensions work universally.

Checkpoint 15 — Difficult / Out-of-the-Box Prompts
Examples: Proverbs as prompts, abstract or philosophical statements (e.g. "The pen is mightier than the sword")

The extra step required: Rephrase the prompt into its real, simplified meaning before planning.

Step-by-step process:

Read the question and circle key words
Ask: Does it require interpretation? If yes → rephrase
Rephrase into plain language (e.g. "The pen is mightier than the sword" → "Peaceful/intellectual means are more powerful than violence")
Apply the 4-step prompt analysis to the rephrased version
Use the 3 dimensions to generate arguments from the simplified meaning
Checkpoint 16 — Editing Your Writing
3-part review checklist:

1. Structure:

Followed the plan?
Argument/discussion point solid?
Connecting words used well?
Template followed?
Introduction has all essential elements?
2. Expression:

Does everything make sense?
Any sentences too long-winded? (Split or cut)
Any sentences too short/abrupt? (Combine)
Right tone used?
Commas and full stops giving clarity?
3. Argument/discussion point development:

Any irrelevant information included? (Remove it)
Any gaps in logic? (Fill them)
Examples specific enough?
Checkpoint 17 — Final Polish
Last 4 checks before submitting:

Spelling — read word-by-word (not sentence-by-sentence) to catch errors
Grammar — especially quotation marks and commas
Connecting words — make sure the right word is used (e.g. "however" vs "therefore" vs "furthermore")
Capitalisation — sentence starts and proper nouns only
Time note: These are minor changes only — no rewriting at this stage.

Key connecting words reference:

Adding: furthermore, in addition, moreover
Contrasting: however, on the other hand, nevertheless
Concluding: therefore, thus, in conclusion, consequently
Checkpoint 18 — Revision & Final Exam
Master process: Plan → Write → Review

Key revision points per checkpoint:

Checkpoint	Key Rule
Write quickly	Follow 3-step process: 25% plan, 60% write, 15% review
Language & tone	Formal tone; reason not exaggeration
Essential parts	Arguments = bulk of marks; strong intro sets the tone
Planning	Streamline; don't waste time
Length	Hit the natural target; more/less = problems
Strong arguments	Use 3 dimensions; arguments must be distinct
Convincing arguments	TEEL with logical progression; no gaps
Supporting arguments	Specific, directly related evidence
Introduction	Topic sentence + stance + signpost (persuasive); both sides + claims (discussion)
Counter-argument	Only include if high quality; 2 sentences
Conclusion	Summarise claims only; no new information
Prompt types	Always do the 4-step analysis regardless of topic
Difficult prompts	Rephrase first, then apply standard process
Editing	Structure → expression → development
Polish	Spelling, grammar, connecting words, capitalisation
Final exam: 30 questions evaluating understanding of all writing strategies across the course.

叙事写作 — 完整 18 Checkpoint 知识体系
模块一：基础认知
CP1 — 什么是高分叙事
三个评判维度：题目关联性 / 顺序与故事线 / 语气塑造
三步流程：规划 → 写作 → 复查
质量优先于完成度（写完但质量差 < 写一半但质量高）
10 大高代价错误：滥用高级词汇、事件太多、只告诉不描绘、逻辑不通、语言华而不实、篇幅失控、内容跑题、无故事线、人物太多
CP2 — 篇幅控制
WA GATE（25 分钟）目标：250 词，4 段，每段 4 句
句子长度诊断：13-15 词 = telling；18-26 词 = showing；26+ 词 = 可能过度
段落原则：长短句混合，不能全短（只 telling）或全长（读者疲劳）
模块二：考试策略
CP3 — 限时写作
时间分配：10% 规划 / 75% 写作 / 15% 复查
WA GATE（25 分钟）：2 min 规划 → 19 min 写作 → 4 min 复查
最大时间陷阱：边写边想结构、中途重写、复查时修改结构
技巧：一句话写出情节，每段只列一个主要点
CP4 — 避免的结构
禁止写：诗歌 / 说明文 / 说服文
禁止：过于复杂的结构 / 过多对话（导致故事线缺失）
原则：考试不是实验场，用经过验证的常规叙事结构
模块三：故事构建
CP5 — 写前规划
结构要素：1 句情节概述 + 每段 1 句关键点（仅 P1–P4）
逆向写法：从结局倒推，每段写"前一件事是什么"
验证清单：主要事件在第 3 段 / 全文只有一个主要事件 / 有解决结尾 / 与题目对齐 / 逻辑推进
CP6 — 回应题目
题目必须是故事核心，而非装饰性出现
图片提示：图中的主要动作 = 主要事件
文字提示：引言出现在文中，但核心事件锚定全文
组合提示：从所有元素中读出整体情绪，分段处理多个组件
原则：情节越简单，越容易与题目深度结合
CP7 — 人物塑造
只用 1–2 个人物
每个人物只突出 1–2 个特质（与故事情绪相符）
要避免：给人物起名字、描述无关特征、引入配角
人物不必是人：动物、地方都可以是人物
CP8 — 故事线生成
三种构思方法：阅读积累 / 个人经历 / 词语联想
词语联想法：写下题目词 → 第一个联想词 → 继续联想 → 形成故事线
关键问题：谁做了这件事？在哪里？有什么意义？遇到什么障碍？结果如何？他人反应？情感如何？
原则：用简单情节，把精力放在写作质量上
模块四：故事发展
CP9 — 故事发展结构
4 段结构：引言 → 铺垫 → 主要事件 → 解决
5 段结构：引言 → 铺垫 → 主要事件前半 → 主要事件后半（戏剧性加强）→ 解决
从结局倒推，确保每段自然衔接下一段
检查：逻辑流畅，段落无跳跃
CP10 — 主要事件（高潮）
定义：出乎意料、戏剧化、重要的事情发生的那一刻
位置：第 3 段（4 段结构）或第 3–4 段（5 段结构）
诊断测试："这个故事是关于什么的？"能用一句话回答 = 主要事件清晰
三大问题：主要事件不可识别 / 完全缺失 / 发展不足
CP11 — 开头与结尾
开头三问：谁在讲故事？故事发生在什么时间？什么语言风格合适？
铺垫段（P2）：描述主要事件前刚刚发生的事，避免时间跳跃
结尾五不：不引入新内容 / 不把主要事件推到结尾 / 不包含多个事件 / 不加不必要的"结束语"（回家吃饭睡觉）/ 不留悬念
结尾的唯一任务：解决主要事件
模块五：表达技巧
CP12 — 写作聚焦（删减）
完成初稿不是结束，真正的提升来自有目的的修改
删除清单：重复描写 / 非核心人物 / 具体名字和年龄 / 品牌名 / 侧面活动（吃早饭、开车上学）/ 与故事无关的物品描写
保留标准：去掉这个内容故事会根本改变 → 留；否则 → 删
原则："More doesn't mean better"
CP13 — 语气塑造
语气 = 描述性词汇 + 感官细节（视觉、声音、气味、触感）
练习法：找一篇新闻稿，加入描述词，感受语气变化
避免：过度使用描述词 / 用词与情感意图冲突（开心场景用"grimacing"）
练习：用"我在跑"分别写出：恐惧 / 愤怒 / 喜悦 / 厌恶 / 悲伤
CP14 — Show, Not Tell
句子级别 4 步法：
写一句"telling"句
找出 3-4 个关键词（代表目标语气）
选择一种感官（最有冲击力的那种）
用 1-2 个最有力的关键词写出描绘句
段落级别平衡：4 句段落 = 2 telling + 2 showing（任意顺序）
原则："1 句深度冲击 > 2 句浅薄描述"
范例段落（从猴架掉落）：
Tell: "One by one, my fingers started to lose hold..."
Show: "My eyes widened and I turned pale..."
Show: "I let out an ear-piercing shriek..."
Tell: "But...that stopped, once I landed with a huge 'thud'..."

CP15 — 出彩故事
差异化手段（各用一次，不过度）：
非常规视角（从动物、物品角度叙述）
明喻/暗喻（约每 250-350 词用 1 个）
意象（战略性地用 1 次）
伏笔（用符号或细节暗示后续事件）
关键：故事线质量 + 想法传达方式 才是真正的差异化来源
模块六：修改润色
CP16 — 修改（编辑）
三项结构检查：有没有按规划写？段落之间是否衔接？结尾有没有解决主要事件？
三项表达检查：全文是否清晰？是否有过长/混乱的句子？是否有过短/突兀的句子？
时态检查：全文保持单一时态，只在叙事逻辑要求时切换
三项测试：
故事是否有秩序感（开头/中间/结尾）？
故事能用一个形容词描述吗（即有没有"感觉"）？
是否符合考试要求？
技巧：用尺子逐词检查（大脑读整句会忽略错误）
CP17 — 最终润色
只做微调（划掉一个字母替换，插入字符）
四项检查：拼写 / 语法（标点、引号、逗号）/ 连接词是否用对 / 大写字母
最终目标：让考官看到"你在乎这篇文章，并且能识别和修正错误"

文本类型模块
CP4 — 叙事文（字面理解）
叙事五要素： 环境 / 主题 / 人物（主次）/ 情节（问题→反应→行动→结果）/ 解决
两种层次： 字面（literal）= 词语直接表意；比喻（figurative）= 词语另有所指
题型：
细节题 → 定位段落，扫描答案
主旨题 → 关键词最多的选项往往是答案
CP5 — 叙事文（比喻理解）
四种比喻手法：
明喻（simile）— "像…一样"
暗喻（metaphor）— "是…"
夸张（hyperbole）— 效果性的过度夸大
拟人（personification）— 赋予非人事物人的特性
答题框架： 定位比喻 → 识别类型 → 理解隐含意义 → 记住答案不是字面意思
CP6 — 叙事文（隐含推断）
触发词： implied / suggest / ascertain / interpret / "how does…feel…?"
答题策略： 找多个"暗示词" → 使用"综合平衡法"，多个线索共同指向答案
原则： 单一关键词不足以得出结论，需要多线索共同倾向一个方向
非虚构文本模块
CP7 — 说明文（明确信息）
特征： 大量细节、客观语气、极少比喻语言
文章结构： 事实信息 + 因果关系 + 推断
细节题： 聚焦段落中明确的因果关系
推断题： 强烈暗示但未明确说明
CP8 — 说明文（隐含推断）
关键词： implied / suggest / ascertain
策略： 找线索 → 找背景信息 → 找关键词倾向 → 评估所有选项
铁律： 只根据文章内容作答，不用自己的背景知识
CP9 — 访谈文本
题型：
观点题 — 区分提问者 vs 被访者的立场
细节题 — 直接信息
推断题 — 字里行间
态度题 — 通过说话方式分析立场
策略： 分析内容、标点、关键词来判断说话者态度
历史与自传模块
CP10 — 历史文本
特征： 事实与叙事解读混合，第三方视角
涵盖全部题型： 事实回忆 / 词汇语境 / 比喻语言 / 作者态度
关键： 历史文章可以是事实与虚构的混合
CP11 — 自传文本
特征： 关于作者自己的生活，第一人称，归类为非虚构
包含"虚构"成分： 记忆失真、选择性叙述、个人诠释
常见主题： 自我发现、身份认知
答题策略： 虚构 + 非虚构双重分析框架
诗歌模块
CP12 — 诗歌结构
诗歌要素：
要素	定义
主题	诗歌讨论的内容
结构	诗节数量
语气（Tone）	声音质感（幽默/愤怒/讽刺）
情绪（Mood）	整体感受（平静/恐惧/困扰）
意象	暗喻/象征/拟人
节奏	节拍、元音重复
押韵	声音模式
对比	相对元素并列
CP13 — 诗歌意义
两层分析： 整体主题 + 单行表达
比喻处理更复杂： "我的手是冰块" 需要 4 步认知处理，远比 "我的手很冷" 更有情感冲击
情绪提取： 识别所有形容词、副词、关键名词 → 综合判断情绪
铁律： 诗歌以比喻语言为主，大多数题目考比喻
漫画模块
CP14 — 漫画（视觉分析）
符号手法： 象征 / 夸张 / 标签 / 类比 / 反讽
视觉设计分析：
留白、大小比例、颜色、位置、角度（向上=优越感，向下=弱势）
面部表情、对话泡泡
CP15 — 漫画（文字分析）
文字拆解三步： 来源识别 → 视觉分析（字体/大小/粗体=有力，花体=创意）→ 分解各部分联想
幽默类型： 夸张 / 不协调 / 意外 / 讽刺 / 反讽 / 刻板印象 / 重复 / 双关
语言技巧模块
CP16 — 标点符号
核心作用： 暂停/语音模式 + 语法结构
常考标点： 句号 / 逗号（歧义消除）/ 引号 / 撇号 / 破折号 / 冒号 / 分号 / 省略号
语气塑造： 句号=终结感；感叹号=兴奋；省略号=犹豫；无标点=开放感

将这个关系延伸到现实生活场景
